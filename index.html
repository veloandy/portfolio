Coding
I have worked in a lot of languages over the years. I appreciate different aspects of different languages. I enjoy learning new ones.
Andy's language-independent approach to coding
  • Make changes in the smallest testable & reviewable increment to move the codebase forward
  • Write automated tests to exercise changes
  • Go through a pull request process to make it easy to unwind provenance of changes in the future
  • Get eyes of others on important changes (and make myself available if others need me)
  • Automate as much as possible so it gets done as a side effect of work (e.g. auto-build artifacts when tag is pushed)
    • Internally facing documentation & utility bash scripts run by humans are smells that point to automation opportunities
